API search


L'interface de programmation (API) search située à cet endroit est une interface qui permet la recherche dans le corpus de la base de donnée des corans en différents langages. L'utilisation de l'API est limitée à 10 000 requêtes par adresse IP*. La technique de recherche utilisée est la méthode TF-IDF qui offre une indexation de contenu relativement rapide et efficace dans la recherche documentaire basée sur différents paramètres.

Exemple d'utilisation de l'API search avec le terme "noble" à l'aide d'une requête AJAX jsonp et jQuery en HTML.

<html>
<head>
	<meta charset="UTF-8">
	<script src="https://vocal.ascmv.net/js/jquery.2.1.3.min.js"></script>
	<script type="text/javascript">
	function afficherlesresultats(term){
	$.ajax({
				url : "https://api.ascmv.net/search/api/mode/classiclimit-ALL/q/"+term+"/limit/300/callback/?",
				dataType:"jsonp",
				jsonp:"callback",
				success:function(data)
						{
						// Récupère le contenu de la div en la sélectionnant
						var resultatsdiv = document.getElementById('resultats');
						// Récupère le nombre de résultats
						var total = Number(data.totalquery)-1;
						// La variable data.aya correspond à la liste des versets
						var tableauaya = data.aya;
						resultatsdiv.innerHTML+= '<p>Total : '+data.totalquery+'</p>';
						// Ci-dessous la boucle permettant l'affichage des résultats dans la div
						for (var i=0; i<total; i++) {
						var preparelesresultats = '<p>Score  :'+data.score[i]+'</p><p>Verset :'+data.aya[i]+'</p>';
						resultatsdiv.innerHTML += preparelesresultats; 
								}
						}
			});
	}
	afficherlesresultats("noble");
	</script>
</head>
<body>
	<div id="resultats"></div>
</body>
</html>

Les paramètres de l'API search


Ci dessous le modèle de requête.

https://api.ascmv.net/search/api/mode/[mode]/q/[requête]/[limit,fuzz]/[nombre]/callback/[texte]

[mode]

classic-ALL
Recherche normale tout langages par terme (méthode TF-IDF) sans paramètre supplémentaire.
classic-AR
Recherche normale en Arabe Uthmani par terme (méthode TF-IDF) sans paramètre supplémentaire.
classic-US
Recherche normale en Anglais (US) par terme (méthode TF-IDF) sans paramètre supplémentaire.
classic-FR
Recherche normale en Français par terme (méthode TF-IDF) sans paramètre supplémentaire.
classiclimit-ALL
Recherche normale avec limite tout langages par terme (méthode TF-IDF) (ajouter la variable limit dans l'url).
classiclimit-AR
Recherche normale avec limite en Arabe Uthmani par terme (méthode TF-IDF) (ajouter la variable limit dans l'url).
classiclimit-US
Recherche normale avec limite en Anglais (US) par terme (méthode TF-IDF) (ajouter la variable limit dans l'url).
classiclimit-FR
Recherche normale avec limite en Français par terme (méthode TF-IDF) (ajouter la variable limit dans l'url).
fuzz-ALL
Recherche fuzzy dans toutes les langues (ajouter le paramètre fuzz dans l'url).
fuzz-AR
Recherche fuzzy en Arabe (ajouter le paramètre fuzz dans l'url).
fuzz-US
Recherche fuzzy en Anglais (US) (ajouter le paramètre fuzz dans l'url).
fuzz-FR
Recherche fuzzy en Français (ajouter le paramètre fuzz dans l'url).
fuzzlimit-ALL
Recherche fuzzy avec limite dans toutes les langues (ajouter les paramètres fuzz et limit dans l'url).
fuzzlimit-AR
Recherche fuzzy avec limite en Arabe (ajouter les paramètres fuzz et limit dans l'url).
fuzzlimit-US
Recherche fuzzy avec limite en Anglais (US) (ajouter les paramètres fuzz et limit dans l'url).
fuzzlimit-FR
Recherche fuzzy avec limite en Français (ajouter les paramètres fuzz et limit dans l'url).

[limit]

nombre
Correspond à la limite des résultats, (limité à 9999). Ce paramètre doît être présent uniquement dans les modes : classiclimit-FR,classiclimit-AR,classiclimit-US,classiclimit-ALL,fuzzlimit-FR,fuzzlimit-AR,fuzzlimit-US,fuzzlimit-ALL.

[fuzz]

nombre
Le paramètre fuzz est un nombre qui correspond a la distance de Levenshtein, par exemple le mot "noble" et "nobel" a une distance de 2 en utilisant cette méthode mathématique, le paramètre fuzz ne peut être utilisé uniquement dans les modes :fuzz-ALL,fuzz-FR,fuzz-US,fuzz-AR,fuzzlimit-ALL,fuzzlimit-FR,fuzzlimit-US,fuzzlimit-AR.

[callback]

variable alphanumérique
Le paramètre callback correspond a la variable de retour JSON (utile pour toute les requêtes cross domain en javascript).


API Hadith

L'interface de programmation (API) hadith permet le renvoi des données relative aux coran. Les données disponibles sont les versets en traduction Française, Anglais(US) et Arabe Uthmani, les sourates (chapitres), ainsi que la récitation par verset.

Ci-dessous un exemple HTML d'utilisation de l'API hadith pour afficher un verset "aléatoirement" en Arable Uthmani.

<html>
<head>
	<meta charset="UTF-8">
	<script src="https://vocal.ascmv.net/js/jquery.2.1.3.min.js"></script>
	<script type="text/javascript">
	// Sélectionne un nombre entier entre 1 et 6236
	var numeroverset = Math.floor(Math.random() * 6236) + 1; 
	function afficherverset(numeroverset){
				$.ajax({
					url : "https://api.ascmv.net/hadith/api/mode/UTH-NUMBERS/num/"+numeroverset+"/callback/?",
					dataType:"jsonp",
					jsonp:"callback",
					success:function(data)
						{
					var resultats = $('#resultats');
					var numsourate = data.numsurateUTH;
					var ayat = data.numayatUTH;
					resultats.html( '<b>Verset numéro :</b> '+numeroverset+'<br><b>Sourate :</b> '+data.numsurateUTH+'<br><b>Verset numero:</b>'+data.numayatUTH+'<br><b> Verset : </b><p class="surateuthman" dir="rtl" lang="ar" style="font-size:200%;    line-height: 200%;">'+data.versetUTH+'</p>');
						}
					});
	}
	afficherverset(numeroverset);
	</script>
</head>
<body>
	<div id="resultats"></div>
</body>
</html>

Les paramètres de l'API hadith


Ci-dessous le modèle des paramètres de la requête.

https://api.ascmv.net/hadith/api/mode/[mode]/[num,aya,surat]/[numéro]/callback/[text]

[mode]

UTH-FULLCHAPTER
Renvoi la sourate (chapitre) en Arabe Uthmani, le paramètre [surat] doît être configuré.
EN-FULLCHAPTER
Renvoi la sourate (chapitre) en Anglais(US), le paramètre [surat] doît être configuré.
FR-FULLCHAPTER
Renvoi la sourate (chapitre) en Français, le paramètre [surat] doît être configuré.
UTH-AYATONLY
Renvoi le verset en Arabe Uthmani, les paramètres [surat] et [aya] doivent être configurés.
EN-AYATONLY
Renvoi le verset en Anglais(US), les paramètres [surat] et [aya] doivent être configurés.
FR-AYATONLY
Renvoi le verset en Français, les paramètres [surat] et [aya] doivent être configurés.
UTH-NUMBERS
Renvoi le verset en Arabe Uthmani, le paramètres [num] compris entre 1 et 6236 doît être configuré.
EN-NUMBERS
Renvoi le verset en Anglais(US), le paramètres [num] compris entre 1 et 6236 doît être configuré.
FR-NUMBERS
Renvoi le verset en Français, le paramètres [num] compris entre 1 et 6236 doît être configuré.
IMG-LOW
Renvoi l'image du verset en Arabe Uthmani Hafs (basse résolution), les paramètres [aya] et [surat] doivent être configurés.
IMG-HIGH
Renvoi l'image du verset en Arabe Uthmani Hafs (haute résolution), les paramètres [aya] et [surat] doivent être configurés.
MP3-ALAFASY
Renvoi la récitation d'un verset par Cheikh Al Afasy, les paramètres [aya] et [surat] doivent être configurés.
MP3-HUSARY
Renvoi la récitation d'un verset par Cheikh Husary, les paramètres [aya] et [surat] doivent être configurés.

[num]

[1-6236]
Doît être une variable numérique comprise entre 1 et 6236 (correspond au nombre de versets).

[surat]

[1-114]
Doît être une variable numérique comprise entre 1 et 114 (correspond au nombre de sourate/chapitres) .

[aya]

[1-*]
Variable numérique entière, correspondant à l'emplacement du verset dans la sourate.

[callback]

[texte]
Variable de retour JSON, alphanumérique.

Limitations

Voilà les limitations des fonctionnalités proposées.

  • - API search - 10 000 requêtes par adresse IP.
  • - 3 langages disponible - Arabe Uthmani, Anglais(US), Français.
  • - Récitation du Coran limitée à 2 cheikh (Husary et Al Afasy).
  • - Moteur de recherche limité aux résultats 3 langues.
  • - Fonctionnalité Fuzz, Limit, et recherche simple match.
